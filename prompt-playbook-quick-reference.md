# Prompt Playbook Quick Reference

这是一份用于日常高频调用的一页式速查表。目标不是完整解释方法，而是让你在真实任务里快速完成输入、组装、调试和评估。默认用法是：先填输入卡，再用组装器生成 Prompt，拿到输出后用评估表判断质量，不理想时直接走调试器。

## 1. 最短 SOP

```text
+先填任务输入卡
+先诊断，再组装 Prompt
+先评估输出，再决定是否调试
+调试时只做最小必要修复
+把有效句子沉淀到元指令库
+```

## 2. 任务输入卡

```text
+1. 你在处理什么内容：
+2. 你希望最终得到什么产物：
+3. 这个产物主要服务于什么：理解 / 决策 / 执行 / 表达
+4. 你最看重的质量目标：深度 / 可执行性 / 批判性 / 清晰度 / 自然文风 / 结构完整
+5. 你最不希望输出出现什么问题：复述 / 模板腔 / 空话 / 伪深度 / 只会赞美 / 不可执行
+6. 输入材料类型：源码 / 题目 / 需求 / 文档 / 数据 / 观点 / 新闻 / 混合材料
+7. 明确边界或约束：
+8. 是否需要最终收敛为结论、建议、规格或行动清单：
+```

## 3. 任务诊断器

```text
+请先不要直接完成任务，而是先完成任务诊断。
+
+基于我提供的信息，输出以下内容：
+1. 任务类型
+2. 输出产物类型
+3. 这个任务真正要解决的问题是什么
+4. 这个任务最关键的质量目标是什么
+5. 这个任务最可能出现的三个失败模式是什么
+6. 最应该强化的三个 Prompt 控制层是什么
+7. 最适合的角色定义方向是什么
+
+最后再给出一句简洁判断：这类任务最怕缺什么。
+```

## 4. Prompt 组装器

```text
+你现在不是直接完成任务，而是根据用户需求为这个任务组装一份高质量 Prompt。
+
+请按以下顺序完成：
+
+第一部分：任务诊断摘要
+- 判断任务类型
+- 判断输出产物
+- 判断最重要的质量目标
+- 判断最危险的失败模式
+
+第二部分：生成 Prompt 的五层结构
+1. 角色层
+2. 问题重定义层
+3. 认知路径层
+4. 质量门槛层
+5. 全局文风层
+
+第三部分：输出最终可直接使用的完整 Prompt
+
+要求：
+1. Prompt 必须由五层构成
+2. 每一层都要服务于结果质量
+3. 如果需求模糊，优先补足最能防止低质量输出的约束
+4. 先给诊断摘要，再给最终 Prompt
+```

## 5. 最小完备骨架

```text
+你是一位【具备特定判断方式的角色】。
+用户的描述是起点，不是边界。先识别问题真正属于什么，再组织回答。
+不要停留在表层解释，必须分析关键约束、代价、失败模式与边界条件。
+最终输出必须解释为什么，指出最关键因素，给出适用边界，并在需要时收敛为可执行判断。
+正文以自然、连贯的叙述性段落为主，结构化格式仅用于天然适合结构化表达的部分。
+```

## 6. 高频增强模板

### 6.1 需求转 Prompt

```text
+我会给你一个原始需求。
+
+你的任务不是直接完成需求，而是先把它升级成一份高质量 Prompt。
+
+请完成以下工作：
+1. 识别任务类型
+2. 识别输出产物
+3. 识别用户最在意的质量目标
+4. 识别最可能出现的失败模式
+5. 基于以上信息生成一份完整 Prompt
+
+Prompt 必须包含：角色定义、问题重定义、认知路径、质量门槛、文风约束。
+
+最后请单独输出“最终 Prompt”，以便我直接复制使用。
+```

### 6.2 提升深度

```text
+不要停留在表层解释或常识复述。
+必须继续分析：
+1. 问题真正的底层结构是什么
+2. 哪些因素决定结果上限
+3. 哪些看起来次要、实际上很关键
+4. 这个判断成立的前提和边界是什么
+```

### 6.3 提升可执行性

```text
+不要把结论停留在分析层。
+最终输出必须收敛为可执行判断，明确：
+1. 下一步该做什么
+2. 在什么条件下做
+3. 成功标准是什么
+4. 出现什么信号时应调整或停止
+```

### 6.4 降低 AI 味

```text
+正文请以自然、流畅、连贯的叙述性段落为主。
+不要使用高频教学腔、总结腔和模板引导语。
+不要把正文写成分点堆砌。
+结构化格式只用于天然适合结构化表达的内容。
+```

## 7. Prompt 调试器

```text
+下面我会给你三个内容：
+1. 原始任务
+2. 当前 Prompt
+3. 当前输出
+
+你的任务不是重写答案，而是先做 Prompt Debugging。
+
+请按以下步骤输出：
+1. 当前输出最主要的失败模式是什么
+2. 这个失败模式最可能来自 Prompt 缺少哪类约束
+3. 应做哪些最小必要修改
+4. 为什么这些修改有效
+5. 输出修复后的 Prompt
+
+注意：
+- 不要大改所有内容
+- 优先做最小必要修复
+- 说明修复的是哪一层控制结构
+```

## 8. 失败模式 -> 修复方向

```text
+如果输出像复述 -> 补 问题重定义 + 认知下钻
+如果输出像模板 -> 补 全局文风约束 + 禁止模板引导语
+如果输出正确但空 -> 补 信息密度约束 + 关键点优先
+如果输出高深但虚 -> 补 边界/验证约束 + 可执行性要求
+如果输出只会赞美 -> 补 批判性指令 + 代价/局限分析
+如果输出很全但不尖锐 -> 补 关键点优先 + 主问题聚焦
+如果输出有洞察但难以行动 -> 补 决策收敛 + 成功标准 + 触发条件
+如果输出局部很好整体不稳 -> 补 独立全局文风层 + 输出节奏约束
+```

## 9. 输出评估表

```text
+输出评估表（每项 1-5 分）
+
+1. 是否真正回答了问题本质，而不只是复述表面需求
+2. 是否识别了关键约束、边界、代价与失败模式
+3. 是否有足够的信息密度，而不是正确废话
+4. 是否给出了明确判断，而不是平均分配篇幅
+5. 是否具备可执行性、可验证性或可决策性
+6. 是否文风自然，避免明显模板腔与 AI 味
+7. 是否整体稳定，而不是局部好、整体散
+
+总分：
+30-35：可直接使用
+24-29：基本可用，但建议做一轮精修
+18-23：存在明显结构缺失，建议进入 Prompt Debugging
+17 以下：不要继续改答案，先重构 Prompt
+```

## 11. 一句话原则

```text
+高质量 Prompt 不是一次写对，而是通过任务诊断、结构组装、结果评估、最小修复和经验沉淀构成的闭环系统。
+```
