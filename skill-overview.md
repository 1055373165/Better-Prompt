# Skill Overview

## Name

Prompt Engineering Skill Package

## Positioning

这是一个将自然语言需求编译为高质量成品 Prompt 的技能包。它的核心定位不是 Prompt 教程，也不是模板仓库，而是一个 Prompt Compiler。用户输入的是一句需求、一个问题或一个模糊目标，系统输出的是一份结构完整、可直接使用、可调试、可评估的 Prompt。

## Core Value

这个 skill package 的价值不在于让用户学习更多 Prompt Engineering 理论，而在于让系统代替用户完成 Prompt 设计。它通过自动识别任务类型、输出目标、质量目标和高风险失败模式，再装配合适的控制模块，帮助用户以极低输入成本获得高质量 Prompt。

## Primary Use Cases

适合以下场景：

- 把一句需求转换成可直接使用的 Prompt
- 把一个模糊问题升级成结构完整的 Prompt
- 对已有 Prompt 做系统化调试
- 对 Prompt 或输出做结构化评估
- 建立从需求到成品 Prompt 的标准工作流

## Trigger Conditions

当用户出现以下意图时建议触发：

- 想把一句需求或问题变成高质量 Prompt
- 想让系统自动补足 Prompt 结构
- 想调试已有 Prompt，而不是重写答案
- 想系统化做 Prompt Engineering
- 想建立标准化 Prompt 工作流

典型触发语包括：

- 帮我把这个需求变成一个高质量 prompt
- 我只给一句话，你帮我生成成品提示词
- 把这个问题升级成可直接使用的 Prompt
- 这个 Prompt 不够好，帮我调试
- 我想系统化做提示词工程

## What This Package Does

- 编译一句需求为成品 Prompt
- 自动补齐角色定义、问题重定义、认知路径、质量门槛和文风约束
- 根据任务类型自动装配控制模块
- 根据失败模式自动给出修复路径
- 提供生成、调试、评估三种主要模式

## What This Package Does Not Do

- 不直接替用户完成原任务本身
- 不保证所有底层模型产出完全相同的质量
- 不替代具体领域知识本身
- 不在缺少上下文时保证绝对正确的事实判断

## Final Principle

这个 skill package 的本质，不是写 Prompt，而是编译 Prompt。只要它能把自然语言需求稳定转换成高质量成品 Prompt，并且输出过程可解释、可调试、可复用，它就完成了自己的使命。
