# Prompt Agent UX & Visual Specification

## 一 文档目标

本文件定义独立 `Prompt Agent` 产品的 UX 与视觉规范。它的目标不是给出一套抽象审美描述，而是明确：

- 产品应如何面向普通用户组织交互
- 如何将“需求翻译型 agent”的价值体现到界面与文案中
- 如何在 UI 视觉上贴近 Apple 式极简与克制
- 如何避免千篇一律的 AI 产品风格
- 如何让整体体验同时满足现代化、专业化与业界最佳实践

本规范应成为后续页面设计、组件设计、文案设计与前端实现的统一基线。

## 二 设计目标

Prompt Agent 的 UX 和视觉设计必须围绕一个核心目标展开：

**让普通用户在没有专业表达能力的情况下，也能感受到产品在“替自己把话说专业、说清楚、说完整”。**

因此，设计目标不是做一个“炫酷 AI 工具”，而是做一个：

- 安静
- 清晰
- 专业
- 克制
- 有判断力
- 让用户愿意长期使用

的个人 agent 工具。

## 三 总体设计哲学

### 1. Apple 风格不是模仿表面，而是模仿原则

我们追求的不是把界面做成某种苹果截图风，而是吸收 Apple 一贯的设计理念：

- 极度克制
- 高级留白
- 信息层级清晰
- 动效有节制
- 功能入口明确
- 内容本身优先于装饰
- 默认体验足够简单，高级能力渐进显露

### 2. 不做典型 AI 产品审美

要避免以下常见 AI 产品套路：

- 过多渐变和泛紫蓝色光效
- 过度强调“智能感”的发光边框
- 聊天气泡式主界面导致工具目标模糊
- 页面信息密度混乱但试图显得先进
- 一眼看上去像另一个 AI 套壳站

### 3. 以“专业工具”而非“娱乐产品”的姿态设计

Prompt Agent 应给用户的第一感觉是：

- 这是一个认真、可靠、专业的工具
- 它理解复杂需求，但不炫耀复杂性
- 它帮助我思考，而不是制造噪音

## 四 核心 UX 原则

### 原则 1：先让用户表达意图，再暴露结构

普通用户不应先面对复杂模式、专业术语和设置项。首屏最核心的体验应是：

- 你想做什么？
- 描述一下你的需求
- 我来帮你把它变成更好的 Prompt

模式和高级设置可以存在，但不应抢在用户表达意图之前压上来。

### 原则 2：系统负责专业化，用户负责表达意图

交互设计必须传递出：

- 用户不需要“说得专业”
- 用户只需要“尽量说出自己想要什么”
- 剩下的结构化与专业化工作由 agent 完成

### 原则 3：默认路径必须极短

首次使用用户应能在极短路径内完成核心行为：

```text
进入页面
-> 输入一句话
-> 选择模式
-> 得到结果
-> 复制使用
```

### 原则 4：结果必须比过程更突出

对于普通用户来说，过程解释当然有价值，但真正最重要的是：

- 成品 Prompt
- 修复后的 Prompt
- 评估结果与下一步建议

因此结果区必须始终优先于说明区。

### 原则 5：系统应持续提出“下一步”

Prompt Agent 作为 agent，不应在结果出来后沉默。系统应主动提供下一步动作，例如：

- 再增强深度
- 再优化语气
- 再提高可执行性
- 自动修最低分项

这既体现 agent 属性，也降低普通用户下一步决策成本。

## 五 信息架构与页面体验原则

## 1. 整体结构

建议采用单工作台结构：

```text
Prompt Agent Workspace
├── Header
├── Intent Input Area
├── Mode Selector
├── Result Workspace
└── Suggested Next Actions
```

### 体验重点

- 输入区是整个页面的心理起点
- 模式切换清晰但不喧宾夺主
- 结果区强调主产物
- 下一步建议始终存在但不过度打扰

## 2. 不采用聊天主界面

虽然很多 AI 产品默认做成聊天形态，但对 Prompt Agent 来说，这会有三个问题：

- 用户会误以为产品目标是“直接回答问题”
- 结构化结果会被聊天气泡打碎
- 专业工具的稳定感和严肃感会被削弱

因此，更推荐工作台式布局，而不是聊天流布局。

## 六 视觉风格规范

### 1. 整体风格关键词

- Minimal
- Calm
- Precise
- Refined
- Professional
- Quiet confidence

### 2. 色彩原则

应使用低噪声、克制、成熟的配色体系。

推荐原则：

- 主色使用低饱和中性色
- 强调色数量严格控制
- 不依赖大面积渐变制造科技感
- 重点通过层级、间距、排版、阴影与边框来建立高级感

更接近的方向应是：

- 白 / 近白 / 浅灰 / 深灰的中性基底
- 搭配少量冷静强调色
- 重点状态使用清晰但不刺眼的反馈色

### 3. 材质与层次

可以适度借鉴 Apple 风格中的以下元素，但必须克制：

- 柔和圆角
- 轻量阴影
- 高质量留白
- 微弱分层
- 低噪声背景

不建议：

- 过强玻璃拟态
- 大面积毛玻璃堆叠
- 过重阴影
- 过密描边

### 4. 排版原则

字体与排版必须体现专业性和清晰度。

推荐排版策略：

- 标题层级明确
- 正文舒展、不过密
- 结果文本使用易复制阅读的版式
- Prompt 本体使用高可读代码/文本容器样式
- 说明文案避免过度营销化口吻

### 5. 图标与视觉元素

图标应：

- 统一线性或简洁风格
- 数量少
- 仅在提升理解时使用

不应让图标承担主要信息层级。

## 七 组件级设计原则

### 1. 输入框

输入框应传递一种低压力感：

- 足够宽敞
- 提示语自然
- 不像专业表单
- 允许用户自然表达

推荐 placeholder 风格：

- 描述一下你想让模型帮你做什么
- 你不需要说得很专业，先把你的想法写下来

### 2. 模式选择器

模式切换应清晰、简洁、低学习成本。

建议使用：

- Tabs
- Segmented control

文案应避免术语过重，可考虑：

- 生成 Prompt
- 调试 Prompt
- 评估 Prompt

必要时在二级说明中补充：

- 帮你从一句需求生成更强 Prompt
- 帮你分析 Prompt 的问题并修复
- 帮你判断 Prompt 质量并打分

### 3. 结果卡片

结果卡片是页面核心。

必须做到：

- 最终产物视觉优先
- 诊断信息清晰但从属
- Copy 按钮显著
- 不要让卡片像营销组件

### 4. Continue Actions 区域

这部分是 agent 感最强的区域，但必须克制。

建议设计成：

- 一组高价值下一步动作按钮
- 动作数量控制在 3-5 个
- 文案具体，不抽象

例如：

- 再增强深度
- 再提高可执行性
- 修复最低分项
- 改成更自然的表达风格

### 5. 评分展示

Evaluate 模式中的评分应：

- 简洁
- 明确
- 易扫读
- 避免把页面做成复杂 dashboard

MVP 阶段不建议做过重图表，可优先使用：

- 分项列表
- 总分突出
- 核心缺陷摘要

## 八 UX 文案原则

### 1. 不假设用户懂 Prompt Engineering

不要默认使用如下文案作为主表达：

- 请输入系统提示词需求
- 请输入提示词控制层
- 请输入认知路径要求

这些语言只适合高级说明，不适合首层体验。

### 2. 文案应降低用户焦虑

用户需要感受到：

- 你不必表达得很专业
- 先把你的意图说出来就可以
- 我会帮你整理成更好的 Prompt

### 3. 文案应传递专业可信感

避免：

- 过度热情
- 夸张智能感表述
- 营销腔
- 轻浮拟人化表达

应追求：

- 冷静
- 清楚
- 明确
- 可靠

## 九 关键页面文案建议

### 首屏标题候选

- 把你的需求，变成更高质量的 Prompt
- 描述你想做什么，剩下的交给 Prompt Agent
- 你负责表达意图，我们负责把它变专业

### 首屏副标题候选

- 即使你不懂专业术语，也能获得更强的 Prompt
- 用自然语言描述你的需求，系统会帮你补齐结构与表达
- 从一句模糊需求开始，生成、调试并优化 Prompt

### Generate 空状态

- 描述一下你想让模型帮你做什么
- 不需要写得很专业，我们会帮你把它整理成更好的 Prompt

### Debug 空状态

- 粘贴你的 Prompt，我们来帮你分析它的问题和可优化空间

### Evaluate 空状态

- 粘贴一个 Prompt 或输出结果，我们来帮你评估它的质量

## 十 Apple 式体验在交互层的体现

如果说视觉层只是表面，那么真正接近 Apple 体验的关键在于交互上的克制与清晰。

### 1. 默认体验足够简单

第一次打开页面时，不应出现太多选项、模块和术语。

### 2. 高级能力渐进呈现

更多细节如：

- 模块使用情况
- 任务类型判断
- 失败模式映射

都可以作为二级信息展示，而不是首屏压满。

### 3. 微动效服务理解，而不是服务炫技

动效应：

- 轻
- 短
- 稳
- 有反馈意义

而不是用来营造“AI 感”。

### 4. 控制感强于惊喜感

用户更需要的是可靠与稳定，而不是每次被“智能感”惊艳。

## 十一 业界最佳实践要求

### 1. 可见系统状态

请求中、成功、失败、可继续优化都必须有清晰反馈。

### 2. 用户控制与自由

用户应能：

- 重新生成
- 继续优化
- 复制结果
- 返回上一步修改输入

### 3. 一致性与标准化

Generate / Debug / Evaluate 三模式的交互框架应一致。

### 4. 识别优于回忆

模式说明、按钮动作、结果结构都应让用户一眼理解，不依赖记忆。

### 5. 降低认知负担

不要要求用户先学模式内部逻辑。产品要先服务，再解释。

### 6. 错误预防优于错误提示

尽量通过输入提示和表单设计降低错误，而不只是等出错后再报错。

## 十二 设计禁区

以下方向应明确避免：

- 把产品做成泛 AI 聊天壳
- 使用俗套科技渐变和发光粒子
- 首页堆砌过多智能卖点
- 用复杂信息密度制造“专业感”
- 用过度术语化文案制造“高级感”
- 用花哨动效替代结构设计

## 十三 设计结论

Prompt Agent 的最佳 UI 方向，不是“更像一个 AI 网站”，而是“更像一个安静、克制、专业的个人工作台”。

它需要同时做到：

- 对普通用户友好
- 对复杂需求有承载力
- 对 Prompt 专家也不过于幼稚
- 视觉上现代且高级
- 气质上专业而不过度冰冷

最终，真正接近 Apple 理念的，不是某种视觉贴皮，而是：

**让复杂能力隐藏在简单、清晰、克制的体验后面。**
